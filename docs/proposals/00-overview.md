# AI Box — Persistent Claude Code via Web Terminal

## Проблема

При разрыве сетевого соединения сессия Claude Code завершается. Контекст диалога теряется, работу приходится начинать заново. Это критично при нестабильном соединении или работе с мобильных устройств.

## Идея

Запустить Claude Code внутри Docker-контейнера, который работает постоянно. Внутри контейнера — tmux-сессия, удерживающая процесс Claude Code живым независимо от состояния сетевого соединения. Поверх — FastAPI, отдающий веб-терминал в браузер. При переподключении браузер просто присоединяется к уже живой tmux-сессии.

## Архитектура

```
Браузер (xterm.js)
    ↕ WebSocket
FastAPI (внутри контейнера)
    ↕ PTY / tmux attach
tmux session "main"
    └── claude (Claude Code CLI)

~/ai/ (volume на хосте)
    ├── .claude/       ← авторизация, настройки Claude Code
    ├── CLAUDE.md      ← системный промпт
    ├── projects/      ← артефакты генерации
    └── .env           ← ANTHROPIC_API_KEY, WEB_TOKEN
```

## Источники компонентов

| Компонент | Источник |
|---|---|
| Docker-образ с Claude Code | [RchGrav/claudebox](https://github.com/RchGrav/claudebox) — референс |
| Веб-терминал (xterm.js + WS) | Текущий проект (app.py + templates/index.html) |
| FastAPI-слой + auth | Реализуется в рамках этого proposal |

## Ключевые свойства

- **Один пользователь** — авторизация через статический токен, БД не нужна
- **Несколько устройств** — можно открыть с телефона и ноутбука одновременно, оба видят одну tmux-сессию
- **Переживает ребилд** — `~/.claude/` лежит в volume, повторная авторизация не нужна
- **Переживает разрыв** — tmux держит сессию, браузер переподключается

## Ограничение

tmux спасает от network drops, но не от исчерпания контекстного окна Claude Code. Когда Claude Code сам инициирует компакцию или завершение — старый диалог уходит. Это фундаментальное ограничение модели, не архитектуры.
